Architecture

Three tiers. Most tools ship one.

Most memory tools give you Tier 1 and call it done. Claude Recall ships all three — and they compound.

TIER 01
Memory
Claude remembers your codebase, decisions, patterns, and known bugs across every session. Nothing is re-explained. The same mistake is never made twice.
TIER 02
Skills
Skills fire automatically from natural language. They score themselves after every use. /evolve reads the scores and patches failing steps — so skills improve without you touching them.
TIER 03
Autonomous
Skills chain into multi-step workflows. Self-healing retries before escalating. Memory auto-pushes after 9pm. Claude acts, not just answers.
Compound Learning Loop

Skills that improve themselves.

Run /learn at End Session. Run /evolve every few sessions. The rest is automatic.

📝
/learn — extract what happened
Reads the session and writes to lessons.md, decisions.md, skill_scores.md, and velocity.md. Scores every skill that fired (Y = needed correction, N = worked first time).
📊
skill_scores.md — binary log
Every skill gets a row: date, what it was used for, whether it needed correction, and exactly which step failed. /evolve uses the "What Failed" column to patch the right line.
⚡
/evolve — patch and cluster
Finds skills with Y scores and rewrites the failing step. Clusters repeated lessons into new reusable skills. Logs every change to skill_improvements.md. The same skill failure never happens twice.
🔁
Next session — better skills
Patched skills load automatically at Start Session. Lessons apply before any code is touched. Rejected approaches in regret.md are never re-proposed.

Autonomous Behaviors

It runs. You work.

These happen without prompting — no commands, no checkpoints.

⛓ Skill Chaining

Skills trigger the next skill on pass or fail. Add one section to any SKILL.md and the chain runs without you.

## Auto-Chain
On pass: → verification-loop
On fail: → debug-topic, retry

fix-bug → verify → smoke-test
↓ fail
debug → smoke-test
🔧 Self-Healing

When a verify step fails, Claude attempts the minimal fix and retries once before escalating to you. Add a Recovery section to any skill to define the fix.

## Recovery
1. touch the changed file
2. wait 5s for reload
3. re-run smoke test
4. escalate if still fails

✓ auto-recovery succeeded
🌙 Auto End Session

The stop hook monitors every response. After 9pm with unsaved memory changes, it auto-commits and pushes to git. Nothing is ever lost — even if you forget End Session.

What Gets Created

Everything ships with setup.py.

Two minutes. Ask a few questions. Done. Commit memory/ and skills/ to your repo — they travel with the code.
